# Interview / Questions

**What is shift-left testing?**

Shift-left means involving QA as early as possible — starting from requirements review, not just after development. A bug found in requirements costs a fraction of what it costs in production. As an SDET I support shift-left by reviewing stories before sprint starts, writing test cases alongside development, and running automated regression on every code push via CI/CD.

**What does QA do in Agile sprints?**

QA participates in all 5 ceremonies. In sprint planning we raise testing concerns and flag missing acceptance criteria. In daily standups we report test progress and blockers. We test features within the same sprint as development — not after. In retrospectives we suggest process improvements. The goal is continuous feedback, not a big-bang testing phase at the end.
